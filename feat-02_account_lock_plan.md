# 不正アクセス対策（アカウントロック機能）の開発計画

## 概要
ログイン失敗が連続して発生した場合に、該当アカウントを一時的にロックすることで、ブルートフォース攻撃等の不正アクセスを防止します。

## 実装内容
1.  **Userエンティティの拡張**
    - `login_attempt_count`: ログイン失敗回数を保持（Integer）
    - `is_locked`: アカウントがロックされているかどうかのフラグ（Integer: 0=未ロック, 1=ロック中）
2.  **ログインロジックの修正 (`LoginValidator`)**
    - ログイン成功時: `login_attempt_count` を 0 にリセットし、`is_locked` が 0 であることを確認。
    - ログイン失敗時: 該当メールアドレスのユーザーが存在する場合、`login_attempt_count` をインクリメント。
    - ロック判定: `login_attempt_count` が 5 回に達した場合、`is_locked` を 1 に更新してロック。
    - ロック中チェック: ログイン試行時に `is_locked` が 1 の場合は認証を通さない。
4.  **アンロック機能の実装**
    - 管理者画面（会員編集）から、ロック状態を解除できるようにする。
    - （オプション）一定時間経過後の自動アンロックについては、要件を確認の上検討する。

5.  **メッセージの追加**
    - アカウントロック時のエラーメッセージを定義。
